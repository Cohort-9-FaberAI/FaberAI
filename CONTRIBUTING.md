# Contributing to FaberAI

## Branch naming

Use the same convention across both `backend/` and `frontend/`:

- `feat/short-description` — new work (e.g. `feat/supabase-integration`, `feat/upload-form`)
- `fix/short-description` — bug fixes (e.g. `fix/upload-bug`, `fix/build-warning`)

## Pull requests

- Open a PR into `main` and fill out the PR template.
- At least one review is required before merging (enforced by branch protection on `main`).
- After your PR is merged, delete your branch to keep the repo clean.
